SECCIÓN 1 — RESUMEN EJECUTIVO
================================================================

App: Servicons Mobile
Plataforma: 100% móvil (React Native + Expo). NO existe dashboard
web ni panel de monitoristas. El custodio es el ÚNICO actor.

QUÉ HACE EL CUSTODIO EN LA APP:
  1. Inicia sesión (Supabase Auth)
  2. Crea bitácoras (wizard de 7 pasos con datos legales del servicio)
  3. Inicia la custodia (GPS + fotos periódicas + botón SOS)
  4. Cierra el servicio (última foto + firma operador + firma custodio)

REGLA DE ORO:
  custodio_id = auth.uid() SIEMPRE.
  La app NUNCA permite insertar un custodio_id diferente al usuario
  de la sesión activa. Sin monitoristas. Sin asignación externa.

-----------------------------------------------------------------
-- TABLA 1: profiles
-- Extiende auth.users con datos del custodio
-----------------------------------------------------------------
CREATE TABLE IF NOT EXISTS public.profiles (
  id          UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  nombre      TEXT NOT NULL,
  email       TEXT,
  celular     TEXT,
  activo      BOOLEAN DEFAULT TRUE,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "usuario_propio" ON public.profiles
  FOR ALL USING (auth.uid() = id);

-- Auto-crear perfil al registrar usuario en Auth:
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, email)
  VALUES (NEW.id, NEW.email);
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();

-----------------------------------------------------------------
-- TABLA 2: bitacoras
-- Una fila = un servicio de custodia completo
-- El campo "formulario" (jsonb) guarda el wizard completo
-----------------------------------------------------------------
CREATE TABLE IF NOT EXISTS public.bitacoras (
  id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  custodio_id             UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,

  -- Columnas resumen (para listar sin parsear JSON)
  nombre                  TEXT,           -- nombre del servicio (paso 1)
  ruta                    TEXT,           -- "origen → destino" o nombre
  unidad                  TEXT,           -- placas del vehículo custodiado
  empresa_contratante     TEXT,

  -- Estado del ciclo de vida
  estado                  TEXT DEFAULT 'pendiente' CHECK (estado IN (
                            'pendiente', 'activo', 'completado', 'cancelado'
                          )),

  -- Formulario completo del wizard (7 pasos)
  formulario              JSONB,

  -- Intervalo de reportes fotográficos (minutos)
  report_interval_minutes INT DEFAULT 15,

  -- Firmas (Base64 data-URL o path en Storage)
  firma_custodio          TEXT,
  firma_operador          TEXT,

  -- Tiempos del servicio
  start_time              TIMESTAMPTZ,
  completed_at            TIMESTAMPTZ,
  created_at              TIMESTAMPTZ DEFAULT NOW(),
  updated_at              TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE public.bitacoras ENABLE ROW LEVEL SECURITY;

-- Solo el custodio dueño puede ver, crear, editar sus bitácoras
CREATE POLICY "dueno_bitacora" ON public.bitacoras
  FOR ALL USING (custodio_id = auth.uid())
  WITH CHECK (custodio_id = auth.uid());

-----------------------------------------------------------------
-- TABLA 3: evidencias
-- Fotos periódicas con GPS durante el servicio activo
-----------------------------------------------------------------
CREATE TABLE IF NOT EXISTS public.evidencias (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  bitacora_id     UUID NOT NULL REFERENCES public.bitacoras(id) ON DELETE CASCADE,
  custodio_id     UUID NOT NULL REFERENCES auth.users(id),
  url_imagen      TEXT,                          -- URL del Storage de Supabase
  latitud         DOUBLE PRECISION NOT NULL,     -- OBLIGATORIO
  longitud        DOUBLE PRECISION NOT NULL,     -- OBLIGATORIO
  observaciones   TEXT,
  timestamp       TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE public.evidencias ENABLE ROW LEVEL SECURITY;

CREATE POLICY "dueno_evidencias" ON public.evidencias
  FOR ALL USING (custodio_id = auth.uid())
  WITH CHECK (custodio_id = auth.uid());

-----------------------------------------------------------------
-- TABLA 4: sos_alerts
-- Alertas de emergencia disparadas desde la app
-----------------------------------------------------------------
CREATE TABLE IF NOT EXISTS public.sos_alerts (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  custodio_id     UUID NOT NULL REFERENCES auth.users(id),
  bitacora_id     UUID REFERENCES public.bitacoras(id),
  latitud         DOUBLE PRECISION NOT NULL,
  longitud        DOUBLE PRECISION NOT NULL,
  estado          TEXT DEFAULT 'activa' CHECK (estado IN (
                    'activa', 'atendida', 'falsa_alarma'
                  )),
  created_at      TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE public.sos_alerts ENABLE ROW LEVEL SECURITY;

CREATE POLICY "dueno_sos" ON public.sos_alerts
  FOR ALL USING (custodio_id = auth.uid())
  WITH CHECK (custodio_id = auth.uid());

-----------------------------------------------------------------
-- STORAGE: buckets para fotos y firmas
-----------------------------------------------------------------
INSERT INTO storage.buckets (id, name, public)
VALUES ('evidencias', 'evidencias', false)
ON CONFLICT DO NOTHING;

INSERT INTO storage.buckets (id, name, public)
VALUES ('firmas', 'firmas', false)
ON CONFLICT DO NOTHING;

-- Política storage: solo el dueño sube/lee sus archivos
-- Ruta obligatoria: {custodio_id}/{bitacora_id}/archivo.jpg
CREATE POLICY "custodio_evidencias_upload" ON storage.objects
  FOR INSERT WITH CHECK (
    bucket_id = 'evidencias'
    AND auth.uid()::text = (storage.foldername(name))[1]
  );

CREATE POLICY "custodio_evidencias_read" ON storage.objects
  FOR SELECT USING (
    bucket_id = 'evidencias'
    AND auth.uid()::text = (storage.foldername(name))[1]
  );

   - Botón "Iniciar custodia":
     → UPDATE bitacoras SET estado='activo', start_time=NOW()
        WHERE id=bitacora_id AND custodio_id=auth.uid()
     → Navega a custody/active?id={bitacora_id}

2. app/(app)/custody/active.tsx — EL CORAZÓN DE LA APP:

   A) TIMER (components/TimerBar.tsx):
      - Cuenta regresiva basada en report_interval_minutes de la bitácora
      - Al llegar a 0: dispara reportarEvidencia() automáticamente
      - Reinicia el timer después de cada reporte
      - Visible como barra de progreso en la parte superior

   B) BOTÓN "REPORTAR AHORA" (manual):
      - Dispara reportarEvidencia() antes de que termine el timer
      - Reinicia el timer

   Función reportarEvidencia():
      i.   Solicitar permiso de cámara (expo-camera) si no lo tiene
      ii.  Solicitar permiso de ubicación (expo-location) si no lo tiene
      iii. Capturar foto con expo-camera (modo imagen)
      iv.  Obtener coordenadas:
             Location.getCurrentPositionAsync({
               accuracy: Location.Accuracy.High
             })
           → destructurar: { latitude, longitude }
      v.   Subir foto a Supabase Storage:
             bucket: 'evidencias'
             path: '{custodio_id}/{bitacora_id}/{Date.now()}.jpg'
           → obtener url_imagen (getPublicUrl o signed URL)
      vi.  INSERT en tabla evidencias:
             { bitacora_id, custodio_id, url_imagen,
               latitud: latitude, longitud: longitude,
               timestamp: new Date().toISOString() }
      vii. Si INSERT exitoso → llamar a n8nService.reportEvidence()

   C) BOTÓN SOS — rojo, grande, siempre visible:
      - Al presionar: mostrar diálogo de confirmación "¿Confirmar SOS?"
      - Si confirma → activarSOS():
          i.  Obtener coordenadas inmediatamente (Accuracy.High)
          ii. INSERT en sos_alerts:
                { custodio_id, bitacora_id, latitud, longitud,
                  estado: 'activa' }
          iii.Llamar a n8nService.sendSOS()
      - Mostrar confirmación "SOS enviado" al usuario

   D) BOTÓN "TERMINAR SERVICIO":
      - Navega a custody/finish?id={bitacora_id}

3. hooks/useLocation.ts:
   - Pedir permisos al montar
   - Exponer getCurrentLocation() → { latitude, longitude }
   - Usar Location.Accuracy.High siempre

4. hooks/useEvidencias.ts:
   - uploadFoto(uri, custodioId, bitacoraId) → url_imagen
   - saveEvidencia(data: Omit<Evidencia, 'id'>) → void
   - Exponer las evidencias de la bitácora actual

SECCIÓN 8 — INTEGRACIÓN CON N8N
================================================================

La app SOLO hace POST a estos endpoints. n8n hace el resto
(WhatsApp, PDFs, alertas). No construir los workflows de n8n.

Archivo: services/n8nService.ts

Las URLs van en .env:
  EXPO_PUBLIC_N8N_BASE_URL=https://tu-instancia-n8n.com

FUNCIÓN 1 — reportEvidence():
  POST /webhook/report-evidence
  Body: {
    bitacora_id: string,
    custodio_id: string,
    url_imagen: string,
    latitud: number,
    longitud: number,
    timestamp: string
  }
  n8n hace: envía foto por WhatsApp

FUNCIÓN 2 — sendSOS():
  POST /webhook/sos
  Body: {
    custodio_id: string,
    bitacora_id: string,
    latitud: number,
    longitud: number,
    timestamp: string
  }
  n8n hace: alerta urgente por WhatsApp con mapa

FUNCIÓN 3 — finishRoute():
  POST /webhook/finish-route
  Body: {
    bitacora_id: string,
    custodio_id: string,
    timestamp: string
  }
  n8n hace: genera PDF del reporte completo y lo envía

REGLA: Si el POST a n8n falla, NO bloquear la app.
       Los datos ya están en Supabase. n8n es secundario.
